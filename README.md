# Laravel OAuth

[![codecov](https://codecov.io/gh/SameOldNick/laravel-oauth/graph/badge.svg?token=knEdg9Q0Ti)](https://codecov.io/gh/SameOldNick/laravel-oauth)

Integrates Laravel Socialite with your app's authentication flow, including account linking, account relinking, registration/login gate checks, and stack-specific response handling.

## Features

- Built-in OAuth flow routes and handlers
- Built-in clients: GitHub, Google, Twitter
- Uses `config/oauth.php` (instead of `config/services.php`) for provider credentials
- Account linking table + model (`oauth_providers`)
- Install command to scaffold app-owned OAuth services and responses
- Contracts for full customization (gate checks, user resolver, registrar, response types)
- Events for connected/disconnected/signed-in lifecycle hooks
- Testing helpers via `OAuth::fake()`

## Requirements

- PHP 8.4+
- Laravel 11.x, 12.x, or 13.x
- `laravel/socialite` 5.27+

## Installation

1. Install the package:

```bash
composer require sameoldnick/laravel-oauth
```

2. Publish config and migrations:

```bash
php artisan vendor:publish --tag=oauth-config
php artisan vendor:publish --tag=oauth-migrations
```

3. Run migrations:

```bash
php artisan migrate
```

4. Scaffold app-level OAuth services and responses:

```bash
php artisan oauth:install --stack=fortify
```

5. Add provider credentials to `.env` (examples):

```dotenv
GITHUB_OAUTH_ENABLED=true
GITHUB_CLIENT_ID=...
GITHUB_CLIENT_SECRET=...
GITHUB_REDIRECT_URI="${APP_URL}/oauth/callback/github"

GOOGLE_OAUTH_ENABLED=true
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_REDIRECT_URI="${APP_URL}/oauth/callback/google"

TWITTER_OAUTH_ENABLED=true
TWITTER_CLIENT_ID=...
TWITTER_CLIENT_SECRET=...
TWITTER_REDIRECT_URI="${APP_URL}/oauth/callback/twitter"
```

6. Ensure the generated app provider is registered.

The install command attempts to add `App\\Providers\\OAuthServiceProvider::class` to `bootstrap/providers.php`. If not added automatically, register it manually.

## Configuration

Package config is in `config/oauth.php`.

### Clients

The `oauth.clients` array controls enabled clients and credentials:

- `enabled`
- `client_id`
- `client_secret`
- `redirect`

By default, the package ships with keys for:

- `github`
- `google`
- `facebook` (config key only unless you add a custom client)
- `twitter`

### Models

`oauth.models` controls:

- Connected account model class (`oauth.models.connected_account.model`)
- User model class (`oauth.models.user.model`)
- User email field (`oauth.models.user.email_field`)

### Routes

Default route setup:

- Prefix: `oauth`
- Middleware: `web`
- Names: `oauth.redirect`, `oauth.callback`

Defaults resolve to:

- `GET /oauth/redirect/{client}`
- `GET /oauth/callback/{client}`

## Usage

### Add OAuth links to your UI

```blade
<a href="{{ route('oauth.redirect', ['client' => 'github']) }}">Continue with GitHub</a>
<a href="{{ route('oauth.redirect', ['client' => 'google']) }}">Continue with Google</a>
<a href="{{ route('oauth.redirect', ['client' => 'twitter']) }}">Continue with X</a>
```

### Add connected account helpers to your user model

```php
<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;
use SameOldNick\OAuth\Concerns\HasConnectedAccounts;

class User extends Authenticatable
{
	use HasConnectedAccounts;
}
```

Then you can call:

```php
$user->hasConnectedAccount('github');
$user->connectedAccount('google');
$user->connectedAccounts()->get();
```

## Scaffolding Options

Use `oauth:install` to generate stack-specific classes your app owns.

```bash
php artisan oauth:install --stack=fortify
```

Supported stacks:

- `fortify`
- `custom`

Useful options:

- `--force` overwrite files
- `--path=` custom output directory (default: `app/OAuth`)
- `--app-namespace=` custom root namespace (default: app namespace)
- `--skip-provider` do not generate app OAuth service provider
- `--skip-registration` do not auto-register provider in `bootstrap/providers.php`

## Customization Model

This package intentionally leaves app behavior to your own classes (generated by `oauth:install`).

Your app provider binds implementations for contracts such as:

- `SameOldNick\\OAuth\\Contracts\\Services\\OAuthGate`
- `SameOldNick\\OAuth\\Contracts\\Services\\OAuthUserRegistrar`
- `SameOldNick\\OAuth\\Contracts\\Services\\OAuthUserResolver`
- `SameOldNick\\OAuth\\Contracts\\Services\\OAuthAuthenticationState`
- `SameOldNick\\OAuth\\Contracts\\Services\\OAuthAccountAssociator`
- Response contracts under `SameOldNick\\OAuth\\Contracts\\Responses\\...`

This allows you to control:

- Who can register via OAuth
- Who can link/unlink providers
- Who can log in via OAuth
- How success/error responses are rendered

## Events

Listen for package events:

- `SameOldNick\\OAuth\\Events\\AccountConnected`
- `SameOldNick\\OAuth\\Events\\AccountDisconnected`
- `SameOldNick\\OAuth\\Events\\OAuthSignedIn`

Example listener registration:

```php
<?php

namespace App\Providers;

use Illuminate\Foundation\Support\Providers\EventServiceProvider as ServiceProvider;
use SameOldNick\OAuth\Events\AccountConnected;
use SameOldNick\OAuth\Events\AccountDisconnected;
use SameOldNick\OAuth\Events\OAuthSignedIn;

class EventServiceProvider extends ServiceProvider
{
	protected $listen = [
		AccountConnected::class => [
			// App\Listeners\HandleAccountConnected::class,
		],
		AccountDisconnected::class => [
			// App\Listeners\HandleAccountDisconnected::class,
		],
		OAuthSignedIn::class => [
			// App\Listeners\HandleOAuthSignedIn::class,
		],
	];
}
```

## Extending With Custom Clients

Register custom clients through the OAuth manager:

```php
<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use SameOldNick\OAuth\Facades\OAuth;
use App\OAuth\Clients\Facebook;

class AppServiceProvider extends ServiceProvider
{
	public function boot(): void
	{
		OAuth::extend('facebook', fn ($app) => $app->make(Facebook::class));
	}
}
```

Your custom client should extend `SameOldNick\\OAuth\\Clients\\Client`.

## Testing

The facade supports faking OAuth providers in tests:

```php
use SameOldNick\OAuth\Facades\OAuth;

OAuth::fake('github', [
	'id' => '1234567890',
	'email' => 'test@example.com',
	'name' => 'Test User',
]);
```

Run package quality checks:

```bash
composer test
composer analyse
composer format
```

## Troubleshooting

- `404` on callback/redirect usually means unknown or unconfigured client.
- Verify `enabled`, `client_id`, and `client_secret` for the client in `config/oauth.php`.
- Ensure OAuth provider redirect URI matches your app callback route exactly.
- If using the scaffold command with `--skip-registration`, register `App\\Providers\\OAuthServiceProvider` manually.
